## 2. Codex 核心结论

- 表面现象不是单一的按钮样式或计时器 bug，而是客户端挂载链路不稳定。
- 首先修正了首页产品态文案、移除了接口预览区，并让 analysis 题允许 `score: 0`。
- 然后修正了题目页状态机：
  - 抽题改为确定性抽题，避免 hydration mismatch。
  - `remainingMs` 首帧按首题时限初始化，不再默认从 `0` 开始。
  - 将答题后的推进副作用移出 `setState` updater，避免 React 开发模式下的不稳定行为。
  - 题目切换时显式重置 `submittedQuestionId`。
- 首页 CTA 增加了更强的显式文字颜色和对齐样式，降低被旧样式或继承样式污染的风险。
- 最关键的真正根因在 `localStorage` 订阅层：
  - `readRecentResult()` 每次 `JSON.parse()` 都返回新对象。
  - 该函数被 `useSyncExternalStore` 作为 snapshot 读取时，会让 React 认为快照持续变化。
  - 结果是触发 `The result of getSnapshot should be cached` 和 `Maximum update depth exceeded`。
  - 这会让首页和游戏页的客户端交互异常，表现为按钮异常、倒计时不动、答题不推进。
- 最终修复是在 `src/lib/storage/local.ts` 中为 recent result 增加基于原始 JSON 的缓存，保证 snapshot 稳定。
